# GURUGURU UI改修モック — A/B/C/D比較レビュー

UI全体改修の方向決めのための4案比較。各案は同一の画面構成・ダミーデータで、デザインだけが異なります。

## 4案のコンセプト

| 案 | 名前 | ねらい |
|---|---|---|
| **A** | Refined Continuity(洗練継承) | 現行のダーク+バイオレットの世界観のまま、余白・階層・質感を洗練 |
| **B** | Pro Density(高密度プロツール) | DaVinci Resolve/Blender系。中性グラファイト+シアン、フラット、情報密度最大化 |
| **C** | Soft Studio(ソフトスタジオ) | Procreate系。ウォームチャコール+アンバー/コーラル、丸み、ゆったり |
| **D** | Layout Shift(レイアウト刷新) | **見た目はA案と同一**、レイアウト構造だけ刷新(左レール・右パネル・フローティングドック) |

比較の読み方:
- **A vs B vs C** → ビジュアル言語(色・質感・密度)の好み
- **A vs D** → レイアウト構造の好み(見た目が同じなので構造だけを判定できる)

## フォルダ構成

```
Docs/UIMock/
├── SPEC.md            … 全案共通の仕様書(要素・ダミーデータ)
├── README.md          … このファイル
├── A/ B/ C/ D/
│   ├── mock.html      … 自己完結モック(ダブルクリックで開ける。ボタンは見た目のみ)
│   ├── home.png       … 画面1: Home(Project一覧・接続設定)
│   ├── project.png    … 画面2: Project詳細(ギャラリー・生成パネル・ツリー)
│   └── modal.png      … 画面3: Assetモーダル(マスク編集モード)
```

`mock.html` をハッシュ付きで開くと単一画面表示になります(例: `mock.html#screen-project`)。

## レビューのしかた(推奨)

見る前に評価軸を意識してください:
1. **一覧性** — 生成結果を見比べやすいか
2. **情報密度** — 必要な情報・操作が一目で見えるか、逆にうるさくないか
3. **疲れにくさ** — 長時間の生成イテレーション作業で目が疲れないか
4. **操作の見つけやすさ** — 次にやりたい操作がすぐ見つかるか

**総合ではなく画面ごとに**判定してください。部分指名も歓迎です(例: 「レイアウトはD、配色はC」「BのタイルデザインだけAに移植」)。
その場合は指名結果を合成したハイブリッド案で2回戦を行います。
