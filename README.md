# GURUGURU

ComfyUIで生成した画像を枝分かれさせながら、試行錯誤の履歴を管理するための実験的な画像生成GUIです。

生成結果を親画像にしてimg2imgを繰り返す「イテレーションツリー」を中心に、プロンプトや生成条件の確認、画像へのペイント・マスク・参照画像の添付、Fountain脚本からの漫画制作などをまとめています。

> [!WARNING]
> 個人の趣味で制作した実験的なアプリケーションです。破壊的な変更や未完成の機能を含む場合があります。

## イテレーションツリー

![GURUGURUの生成画面とイテレーションツリー](Docs/assets/iteration-tree.png)

画面下部のツリーでは、各生成ラウンドをノード、親画像から派生した生成を枝として表示します。ノードを選択して生成条件や結果を確認し、任意の画像から新しいブランチを作成できます。枝の色はデノイズ強度に応じて変化するため、元画像からどの程度変化させたかも追いやすくなっています。

## 主な機能

- txt2img / img2imgの生成履歴をツリーで管理
- ペイント、inpaintマスク、ポーズControlNet、参照画像の添付
- プロンプト、LoRA、seed、サンプラーなどの生成条件管理
- Fountain脚本の取り込み、コマ割り、吹き出し配置、漫画ページ生成
- PNG、PPTX、OpenRaster（ORA）への書き出し

## 必要なもの

- [Bun](https://bun.sh/) 1.3.14以上
- 起動済みのComfyUI（既定の接続先は `http://127.0.0.1:8188`）
- 使用するワークフローに応じたComfyUIのモデルとカスタムノード

## 起動方法

```powershell
git clone <このリポジトリのURL>
cd guruguru
bun install
bun run build
bun run start
```

ブラウザで [http://127.0.0.1:5177](http://127.0.0.1:5177) を開きます。ComfyUIを別のURLで動かしている場合は、GURUGURUの設定画面でBase URLとWebSocket URLを変更してください。

開発時は、ソース変更時の自動ビルド・再起動・ブラウザリロードを行う次のコマンドを利用できます。

```powershell
bun run dev
```

ユーザーデータや生成画像はリポジトリ内ではなく、OSのユーザーデータディレクトリ（Windowsでは `%LOCALAPPDATA%\GURUGURU`）へ保存されます。

## 経緯

画像生成時のimg2imgイテレーション履歴を管理したい、というアイデアからGUIツールの制作を始めました。

基本的な画像生成機能が一段落した後、「Fountain脚本から漫画を作成する機能」の開発に着手しました。本来は脚本からコマ割りなどのネームへ変換する工程にローカルVLMを使用する想定でしたが、Codexをオペレーターとすることで一定の形になったため、この趣味プロジェクトの開発に一区切りをつけています。

詳しい開発・運用上の注意は [操作メモ.md](操作メモ.md)、内部設計は [Docs](Docs/README.md) を参照してください。
