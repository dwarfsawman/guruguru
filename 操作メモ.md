# 操作メモ

GURUGURU の操作方法や注意点を追記するためのメモです。大きな仕様書ではなく、実際に開発・運用するときに必要な手順と落とし穴を残します。

## 開発時の基本操作

- 依存関係は `npm install` で用意します。
- ビルドは `npm run build` を使います。
- 開発起動は `npm run dev` を使います。これはビルド後に `dist/server/index.js` を起動します。
- 構文チェックを含む確認は `npm run check` を使います。
- 実行には Node.js 24 以上が必要です。

## データ保存場所

- 通常実行時のユーザーデータは OS のユーザーデータディレクトリに保存されます。Windows では `%LOCALAPPDATA%\GURUGURU` が既定です。
- `GURUGURU_DATA_DIR` を指定する場合は、必ずこのリポジトリ外のパスにします。
- `app.db`、生成画像、プロンプト、ワークフロースナップショット、サムネイルなどのランタイムデータはリポジトリに置きません。
- リポジトリ直下や `data/` に実運用データベースを作らないでください。

## テスト・検証時の注意

- データベースを初期化するコマンドやテストでは `GURUGURU_TEST_DB=1` を設定します。
- テスト用データの保存先を明示したい場合は `GURUGURU_TEST_DATA_DIR` を使います。
- 本番のユーザーデータディレクトリや実運用の `app.db` を、テスト・調査・要約目的で読み込まないでください。
- コマンドがリポジトリ内のデータディレクトリを拒否した場合は、外部のテスト用ディレクトリを指定してやり直します。

## ドキュメント運用

- 新しい操作手順、注意点、調査で分かった落とし穴はこのファイルに追記します。
- リポジトリに関する Markdown を更新したときは、変更履歴にも要点を追記します。
- 古い大きな仕様書を復活させるのではなく、実際に役立つ運用メモとして短く保ちます。
